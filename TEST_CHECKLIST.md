# VibePetCore Test Checklist

Короткий чеклист перед реальными тестами на сервере. Идти сверху вниз, без добавления новых фич во время проверки.

## 0. Release Gate 2.6.26

Уже доказано live smoke:
- `ПКМ` по Источнику с core в offhand открывает GUI и не запускает параллельный призыв.
- GUI `quest accept/turn-in` реально списывает предметы и начисляет очки.
- GUI `forge` реально тратит donor eggs.
- GUI `Source box` реально выдаёт награду.
- GUI `evolution` реально тратит stage materials и поднимает `E1 -> E2`.
- GUI `repair` реально тратит 1 тотем и чинит `durability +1`.

Наблюдённый факт после успешной эволюции:
- `level` и `bond` сбрасываются для новой стадии.
- Это подтверждено текущим кодом `CoreProgressionAPI.tryEvolve(...)`, а не считается регрессом smoke.

Важно про `/vpc admin audit`:
- при активной `WARPED_BUTTON` audit сейчас шумит строкой про `legacy active-buttons: 1`.
- для текущего релизного gate это считать admin-noise, а не автоматическим fail.
- fail только если есть дубли `petId`, пропажа ядра, runtime/data mismatch, stacktrace или реальные потери ресурсов.

Подготовка для тестера:
- открыть тестовый сервер с правами op;
- поставить Источник: `/vpc source set`;
- если нужен новый core: `/vpc admin giveegg <ник> wolf common`;
- держать тестируемое ядро во второй руке;
- после каждого сценария смотреть `/vpc debugpet`, `/pet info`, серверный лог и факт расхода предметов.

## 0.1 Final Stack Gate

Это уже не bug-hunt, а финальный server-stack проход для 2-3 тестеров.

Тестер 1: Source / Economy
- Команды:
  - `/vpc source set`
  - `/vpc admin giveegg <ник> wolf common`
  - `/vpc admin addpoints <ник> 200`
- Шаги:
  - положить core во вторую руку;
  - ПКМ, призвать питомца;
  - `/pet menu main`;
  - слот `16` -> `22` открыть Source box;
  - слот `10` -> вкладка `gather` -> взять первый gather quest;
  - с другим ядром в `main hand` проверить `/pet quest list evolution` или вкладку `evolution`: выбранный питомец и стадия всё равно должны идти по активному/offhand ядру;
  - выдать нужный предмет и сдать тем же GUI-слотом.
- PASS:
  - нет второго случайного призыва при клике по Источнику;
  - box даёт ровно одну награду;
  - `/pet quest` и Source quest GUI не перескакивают на чужое/main-hand ядро;
  - quest списывает нужные предметы и даёт очки один раз;
  - нет жалоб на “непонятно куда нажимать”.

Тестер 2: /pet UX / Repair
- Команды:
  - `/vpc admin giveegg <ник> wolf common`
- Шаги:
  - открыть `/pet`;
  - проверить, что слот `13` читается как вход в рост/эволюцию;
  - слот `17` открывает help;
  - `/pet help summon` пишет про `offhand`, первый summon `3 сек`, обычный `2 сек`;
  - слот `24` на повреждённом ядре чинит ровно `+1`;
  - во время `repair` держать второе чужое ядро в `main hand`: ремонт всё равно должен попасть в активное/показанное ядро из `/pet`;
  - при двух `wolf` ядрах: пустое чужое ядро не должно принять активного волка другого `petId`;
  - слот `36/38` меняет distance без визуальной лжи.
- PASS:
  - меню читается без объяснения кода;
  - help не врёт про summon hand/timer;
  - repair не ест лишние тотемы;
  - чужое ядро в `main hand` не перехватывает personal `/pet` repair;
  - второе пустое ядро того же типа не затирает другого питомца;
  - не появляется дубль ядра/кнопки.

Тестер 3: Forge / Evolution / MSPT
- Команды:
  - `/mspt`
  - если `mspt` недоступен: `/spark tps`
- Шаги:
  - рядом с Источником прогнать summon -> forge -> evolution;
  - во время `forge` держать другое ядро в `main hand`: кузня всё равно должна работать по активному/offhand ядру, а не тихо перескочить на второе;
  - открыть вкладку `evolution` в квестах Источника с другим ядром в `main hand`: GUI и блокировка квеста должны смотреть на активное/offhand ядро, а не на чужое;
  - в это же время второй игрок открывает `/pet` и Source GUI;
  - смотреть 1-2 минуты на MSPT/TPS во время кликов.
- PASS:
  - нет sustained MSPT > `35` и нет серии пиков к `50+` только из-за VibePetCore сценариев;
  - `forge` не перескакивает на чужое/main-hand ядро и не тратит donor eggs не в тот core;
  - `evolution quest` не блокируется и не показывает чужую стадию из `main hand` ядра;
  - forge/evolution не ломают GUI соседнего игрока;
  - нет stacktrace в логах.

## 1. Призыв и предмет

- Выдать новое ядро питомца.
- Положить ядро во вторую руку.
- Нажать ПКМ и дождаться призыва.
- Проверить, что ядро стало активной кнопкой.
- С активной кнопкой во второй руке проверить, что в инвентаре нет обычного пустого ядра как ложного сигнала.
- Во время таймера повторно нажать ПКМ: отсчёт не должен сбрасываться, новые эффекты не должны спамиться.
- При искусственном сбое сохранения во время призыва: ядро остаётся заполненным, активная кнопка не появляется, runtime-питомец не остаётся активным.
- Снять предмет из второй руки: питомец должен скрыться.
- Вернуть предмет во вторую руку и снова призвать.
- Перейти в creative и spectator: питомец должен убраться обратно в ядро, активная кнопка не должна оставаться живой.
- В creative попробовать перетащить/клонировать/поменять местами ядро: не должно оставаться дубликатов одного `petId`.

## 2. Режимы питомца

- `/pet call`: питомец возвращается ближе.
- `/pet stay`: питомец держится рядом, но не считается оставленным дома.
- `/pet follow`: питомец снова следует.
- `/pet defense`: боевые реакции включаются/выключаются.
- `/pet autoloot`: работает только у подходящих питомцев.
- `/pet position` и кнопка `Позиция`: циклично меняют место питомца рядом с игроком.
- В GUI `/pet` позиционный контроллер меняет только место и дистанцию; режимы `рядом` / `следует` проверять командами `/pet stay` и `/pet follow`.

## 3. Движение

- Пройти по траве, ступенькам, склону, воде рядом, узкому проходу.
- Проверить, что питомец прыгает/обходит мелкие блоки.
- На Э1 проверить обычный блок перед питомцем: он должен прыгать заранее и перепрыгивать, а не упираться в грань.
- Проверить позиции 0-7: спереди, спереди справа, справа, сзади справа, сзади, сзади слева, слева, спереди слева.
- Проверить дистанцию: кнопки `Ближе` / `Дальше` реально меняют расстояние следования.
- Убежать далеко: питомец должен безопасно вернуться или телепортироваться рядом.
- Сменить мир и вернуться.

## 3.1. Живость

- Постояв AFK 20-40 секунд, проверить короткие действия питомца: смотрит, отдыхает, любопытствует, изучает предмет/траву.
- Держать еду/редкий ресурс в руке: питомец иногда подходит и изучает предмет.
- Смотреть на питомца: action bar показывает короткую подсказку и базовый статус.
- Проверить пассивные эффекты на Э1: должен быть один ключевой эффект коротким окном, потом пауза, без постоянного спама обновления.
- Проверить Э4-Э5: основные эффекты могут держаться почти постоянно, но не больше 3 эффектов одновременно.
- Открыть help по каждому питомцу: вместо абстрактной роли должны быть `Польза` и конкретные бафы по эволюциям.
- Поставить рядом двух игроков с питомцами: иногда появляются мирные взаимодействия без драки питомцев.
- Один раз ударить чужого питомца: прямого урона по entity нет, питомец должен насторожиться.
- Повторно спровоцировать питомца или продолжить PvP между владельцами: должна открыться короткая controlled-схватка питомцев без бесконечной цепочки агра.

## 4. Прогресс

- Покормить питомца подходящей едой.
- При полной сытости обычная еда не должна тратиться; HP не должен расти сразу от еды.
- После урона HP должен восстанавливаться постепенно только при наличии сытости.
- Проверить лимит: еда не должна давать больше 20% прогресса уровня за раз.
- Открыть GUI эволюции и проверить, чего не хватает по связи, ресурсам и квестам стадии.
- Кликнуть активного питомца предметом эволюции (`NETHER_STAR`, `ECHO_SHARD`, `DIAMOND`): эволюция не запускается, предмет не списывается, появляется короткая подсказка про `/pet` или Источник.
- Положить ресурсы эволюции в инвентарь игрока и в рюкзак питомца.
- Проверить кузню редкости в Источнике: при искусственном сбое сохранения donor-яйца и редкость ядра должны восстановиться, ложный успех не показывается.
- Проверить ремонт ядра: при искусственном сбое сохранения тотем, прочность, сытость и штраф смерти должны восстановиться, ложный успех не показывается.
- Для Э2 проверить порог `ур.3`, `связь 4/10` и `1 из 5` заданий стадии.
- Для Э3 проверить порог `ур.5`, `связь 6/10` и `1 из 5` заданий стадии.
- Для Э4 проверить порог `2 из 5` заданий стадии.
- Для Э5 проверить порог `3 из 5` заданий стадии.

## 5. Рюкзак и лут

- `/pet vault` открывает рюкзак активного питомца.
- Предметы остаются после закрытия/открытия.
- У лутера образец в рюкзаке задаёт фильтр.
- Проверить автоподбор у Allay, Fox, Bee, Bat и Parrot.
- Автоподбор не трогает запрещённые предметы.

## 6. Смерть и рестарт

- Умереть с активным питомцем.
- Проверить штраф восстановления в лоре/баре.
- Перезайти на сервер.
- Перезапустить сервер.
- Проверить, что питомец, ядро и активная кнопка не потерялись.

## 7. Админка

- `/vpc admin inspect` показывает понятное состояние.
- `/vpc admin fixpet` помогает при пропавшем runtime-питомце.
- `/vpc DANGER-DELETE <player>` требует `/vpc confirm` и чистит питомцев игрока.
- `/vpc DANGER-DELETE All` использовать только на тестовом мире или перед осознанной чисткой.
- `/vpc source set` ставит Источник питомцев на текущую точку.
- `/vpc source info` показывает точку Источника питомцев.
- Рядом с Источником открывается `/pet box open`, и вокруг Источника проигрывается каст-эффект.
- При искусственном сбое сохранения во время открытия Источника: яйцо не выдаётся, очки/cooldown/pity не продвигаются, игрок видит сообщение повторить позже.
- `/vpc lootbox set <arcane|nature|royal>` ставит свой алтарь лутбокса на текущую точку.
- Клик по самому алтарю лутбокса открывает коробку без ручного блока.
- С ядром питомца во второй руке нажать ПКМ по Источнику: открывается GUI Источника, призыв питомца не стартует.
- Быстрый двойной клик по Источнику/алтарю не должен выдавать два яйца за один короткий дубль события.
