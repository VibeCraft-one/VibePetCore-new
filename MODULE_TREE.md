# VibePetCore: Quick Map

Короткий вход для Codex/ChatGPT перед правками. Не раздувать файл: обновлять только важные правила, слабые места и последние крупные изменения.

## Mini Prompt

Ты работаешь с Paper/Purpur-плагином VibePetCore. Сначала опирайся на эту карту, потом читай только нужные файлы. Не придумывай API Bukkit/Paper, методы проекта или механику без проверки в коде. Не делай тяжёлые тики, массовые сканы мира, бесконечные задачи, синхронные IO-операции в частых событиях и логику GUI внутри бизнес-сервисов. Любое поведение для игроков должно быть простым, русскоязычным и проверяемым. Если меняется поведение, подними версию в `build.gradle.kts`, добавь `MIGRATION-*.txt` и собери `gradlew.bat processResources compileJava jar`. Перед ручными серверными тестами держись `TEST_CHECKLIST.md`; не добавляй новые фичи во время прохода чеклиста.

Windows note: PowerShell иногда показывает русский текст кракозябрами, хотя файлы нормальные в UTF-8. Для чтения используй `Get-Content -Encoding UTF8` и при необходимости `$OutputEncoding = [System.Text.UTF8Encoding]::new(); [Console]::OutputEncoding = [System.Text.UTF8Encoding]::new()`.

## Module Tree

- `core` - запуск плагина, команды, регистрация событий, lifecycle модулей.
- `config` - баланс и настройки; не хранит игровую логику.
- `player` - `PlayerData`, `OwnedPetData`, storage; источник правды по прогрессу.
- `egg` - ядро питомца, активная кнопка, призыв через руку, защита от vanilla-spawn яиц.
- `pet` - runtime-питомец: spawn/despawn, движение, бой, action bar, реакции.
- `progression` - опыт, кормление, уровни, эволюция.
- `gui` - только отображение и вызов сервисов; не владеть бизнес-логикой.
- `quest`, `box`, `economy` - задания, лутбоксы, награды.
- `task` - периодические тики; держать лёгкими и предсказуемыми.

## Player Pet Rules

- Активный питомец привязан к игроку; механики "оставить дома" сейчас нет.
- 7 кнопок GUI питомца: `К себе`, `Режим движения`, `Позиция`, `Рюкзак`, `Эволюция`, `Автоподбор`, `Защита игрока`.
- После призыва ядро превращается в активную кнопку; во второй руке она открывает меню питомца через ПКМ с таймером 2 секунды.
- Разовые действия: `К себе`, `Рюкзак`, `Эволюция`.
- Режим движения: одна кнопка переключает `держаться рядом` / `следовать`.
- Позиция: одна кнопка циклично меняет место питомца рядом с игроком; без отдельной кнопки на каждую сторону.
- Живость питомца симулируется в `RuntimePet` через лёгкие ambient-события, без включения ванильного AI.
- Action bar может кратко объяснять событие: `Лиса: изучает цветок | [HP ...]`.
- PvP между питомцами не использует обычный урон entity: сначала идёт PvP владельцев, потом предупреждение/эскалация, и только затем короткая controlled-схватка питомцев.
- Постоянные переключатели: `Автоподбор`, `Защита игрока`.
- Команды-дубли: `/pet call`, `/pet stay`, `/pet follow`, `/pet vault`, `/pet autoloot`, `/pet defense`, `/pet position`, `/pet evolve`.

## Weak Spots

- Самая хрупкая связка: `PlayerData` <-> предмет во второй руке <-> runtime-питомец.
- Любые creative/spectator сценарии проверять отдельно: там питомец теперь обязан сворачиваться обратно в ядро, без живого runtime.
- Защищённые зоны типа `/spawn` могут резать `CreatureSpawnEvent`; для питомцев есть отдельный spawn-guard, но это всё равно проверять руками на сервере.
- Проверять сценарии: join, quit, respawn, смена мира, creative/survival, рестарт сервера, снятие/возврат ядра, старые яйца.
- Старые или дюпнутые поколения чистятся через `/vpc DANGER-DELETE All` или `/vpc DANGER-DELETE <player>`.

- Всегда держи русские строки в UTF-8 без BOM.
- Не перезаписывай большие файлы с русским текстом без нужды: лучше точечный патч и сразу `compileJava`.
- Если нужно чинить текст массово, начинай с `CODEX_CONTEXT.md` и `TEMP_MOJIBAKE_RU_CHECKLIST.md`, а не с кода.
