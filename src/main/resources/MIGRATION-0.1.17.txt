VibePetCore 0.1.17 - производительность и заметки на следующий этап

1. Сохранение данных игроков теперь работает по dirty-схеме: интервал и ручной save пишут только изменённых игроков.
2. Offhand-синк больше не крутится глобальным обходом всех игроков каждые 10 тиков. Обновление теперь ставится в очередь по событиям инвентаря/свапа/респавна/смены мира.
3. В runtime питомцев и utility-логике сокращено число nearby-сканов: часть двойных проходов схлопнута, добавлен кэш nearby-энтити на тик.
4. Старые данные игроков дополнительно руками мигрировать не нужно: формат players.db / json fallback остаётся совместимым.
5. ВАЖНАЯ ПОМЕТКА НА СЛЕДУЮЩУЮ МИГРАЦИЮ: обязательно вернуть механику компактных кнопок во второй руке, когда ядро/яйцо питомца визуально заменяется на маленькую кнопку.
   Это критично для UX: почти все полноценные текстуры предметов во второй руке слишком большие и мешают игрокам, а цветные кнопки делают процесс заметно приятнее.
6. Если на каком-то этапе offhand снова показывает полное яйцо вместо кнопки, это не финальное поведение и его надо считать regression/TODO для ближайшего этапа.
